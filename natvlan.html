🔵  Bahagian 1 — NAT & PAT (Network/Port Address Translation)
🏠  Kenapa Kita Perlu NAT? — Analogi Rumah
Bayangkan kau tinggal dalam flat. Flat tu ada satu alamat pos je — contoh 203.0.113.5. Tapi dalam flat ada 10 bilik — setiap bilik ada nombor bilik sendiri seperti 192.168.1.2, 192.168.1.3 dan sebagainya.

Bila jiran luar hantar surat, dia hantar ke alamat flat (public IP). Penjaga flat (router) yang akan forward surat ke bilik yang betul berdasarkan siapa yang minta.

Itulah NAT — sorokkan banyak IP private di belakang satu IP public.

📦  NAT vs PAT — Apa Beza?
NAT (basic) — map satu IP private → satu IP public. Contoh: 5 PC = perlu 5 IP public. Boros IP.

PAT (Port Address Translation) — map BANYAK IP private → SATU IP public, bezakan menggunakan nombor port. Inilah yang rumah dan pejabat guna sekarang.

PAT juga dipanggil NAT Overload — sebab satu IP public di-"overload" untuk ramai user.

SCENARIO — TEST FAHAM KAU
❓ "A company wants all 200 employees to share a single public IP address to access the internet." — Apa yang kena configure?
→ PAT (NAT Overload)
Sebab: banyak private IP → satu public IP, bezakan guna port number.
❓ "A firewall admin is mapping a server's internal IP 10.0.0.5 to external IP 203.0.113.10 for public access." — Apa function ni?
→ NAT (Static NAT / VIP)
Sebab: satu-satu mapping, supaya orang luar boleh reach server dari luar.
❓ "Which technology hides internal IP addresses from the internet?" — NAT atau PAT?
→ NAT (broad answer) atau PAT jika context ramai users
Kedua-dua sorokkan private IP. Tapi kalau ada "many devices" atau "single IP" dalam soalan → PAT.
🟣  Bahagian 2 — VLAN (Virtual LAN)
🏢  Kenapa Kita Perlu VLAN? — Analogi Pejabat
Bayangkan satu pejabat besar — semua orang, Pemasaran, HR, IT, dan Kewangan, dalam satu bangunan dengan satu switch besar.

Tanpa VLAN: semua orang dalam satu network. Orang Pemasaran boleh cuba akses fail Kewangan. Broadcast dari satu PC flood ke semua orang — perlahan, tak selamat.

Dengan VLAN: walaupun fiziknya semua dalam satu switch, kita wujudkan "dinding logik" — Kewangan dalam VLAN 10, HR dalam VLAN 20, IT dalam VLAN 30. Mereka tak nampak traffic antara satu sama lain walaupun kabel dalam switch sama.

VLAN = Partition logik dalam switch. Fizik sama, tapi logik berasingan.
JENIS-JENIS VLAN — FAHAM SETIAP SATU
Data VLAN
Biasa: VLAN 10, 20, 30...
Untuk traffic biasa — PC, laptop, printer.

Ini VLAN yang paling biasa. Setiap department ada Data VLAN sendiri.
Bila dengar "VLAN" dalam soalan tanpa qualifier = biasanya Data VLAN.
Voice VLAN
Biasa: VLAN 100, 150...
Khas untuk VoIP phones sahaja.

Kenapa kena asingkan? Supaya phone dapat QoS priority — kalau voice traffic bercampur data, suara jadi putus-putus (jitter).
"Phone cannot receive calls" + port issues → check Voice VLAN assignment.
Management VLAN
Biasa: VLAN 99, VLAN 1 (default)
Untuk manage switch/router — SSH, Telnet, SNMP, web GUI.

Best practice: asingkan dari user traffic. Hadkan siapa boleh akses.
Admin access ke switch patut guna dedicated Management VLAN.
Native VLAN
Default: VLAN 1 (tukar untuk security)
Traffic yang tidak ada tag 802.1Q pada trunk port akan masuk Native VLAN.

Macam "VLAN default" untuk trunk — kalau frame tak ada label, masuk sini.
VLAN hopping attack guna Native VLAN mismatch. Tukar dari VLAN 1!
Default VLAN
Selalunya VLAN 1
VLAN yang semua port masuk secara default bila switch baru keluar dari kotak.

Tak boleh delete VLAN 1. Tapi best practice — jangan guna untuk user traffic.
Switch baru → semua port dalam VLAN 1. Kena reassign ke VLAN yang betul.
Black Hole VLAN
Biasa: VLAN 999 atau unused VLAN
VLAN "buangan" — port yang tidak digunakan dimasukkan sini dan disabled.

Security measure — prevent unauthorized device dari connect ke unused port.
"Disable unused switchports" → assign ke Black Hole VLAN.
🔌  Access Port vs Trunk Port — Beza Penting
Access Port — port untuk end device (PC, printer, phone). Port ni hanya tahu satu VLAN. Frame yang masuk/keluar tak ada VLAN tag — device tak perlu tahu dia dalam VLAN mana.

Trunk Port — port untuk hubung switch ke switch atau switch ke router. Port ni boleh bawa banyak VLAN serentak. Guna 802.1Q tagging untuk label setiap frame dengan VLAN ID dia.

Analogi: Access port = jalan sehala masuk satu kawasan. Trunk port = lebuh raya yang bawa banyak jenis kenderaan, setiap kenderaan ada nombor plate (VLAN tag).

🏷  802.1Q Tagging — Macam Mana Tag Berfungsi
Bila frame masuk trunk port, switch tambah tag 4-byte dalam frame header — tag ni mengandungi VLAN ID (1–4094).

Bila frame sampai switch lain, switch tu baca tag, tahu frame tu untuk VLAN mana, then buang tag sebelum forward ke access port.

End device (PC, phone) tak nampak tag langsung — switch yang handle semua ni.

Native VLAN = satu-satunya VLAN yang frame dia tak kena tag pada trunk port. Ini yang boleh menyebabkan VLAN hopping attack kalau misconfigured.
⚠ VLAN Exam Traps — Yang Selalu Keliru
Voice VLAN config Port phone guna access mode, bukan trunk. Tambah switchport voice vlan X — port tu auto support 2 VLAN (data + voice).
PC + Phone satu port PC connect ke phone, phone connect ke switch — satu port boleh handle data VLAN + voice VLAN serentak. Phone tag sendiri traffic dia.
Native VLAN mismatch Dua switch trunk port kena ada Native VLAN sama. Kalau berbeza → VLAN hopping boleh berlaku.
SVI routing Nak route antara VLAN kena ada Layer 3 switch atau router. SVI (Switch Virtual Interface) = logical interface untuk routing antara VLAN dalam L3 switch.
Workstation moved "User moved to different area, can access internet but not server" → Wrong VLAN assignment pada port baru.
IP helper DHCP request tak boleh cross VLAN boundary sendiri. Guna IP helper address pada router/L3 switch untuk forward DHCP request ke server lain subnet.
SCENARIO — VLAN
❓ "A VoIP phone is plugged into a port but cannot receive calls. The PC connected through the phone works fine." — Apa masalahnya?
→ Port tu tak ada Voice VLAN configured
PC okay (Data VLAN ada) tapi phone tak dapat VLAN dia sendiri. Fix: tambah switchport voice vlan 100
❓ "After moving to a new desk, a user can browse internet but cannot access the file server in VLAN 30." — Kenapa?
→ Port baru tu dalam VLAN yang salah
Port lama VLAN 30 (sama subnet dengan server). Port baru mungkin dalam VLAN 10 atau VLAN 1. Internet jalan sebab ada routing keluar, tapi file server dalam VLAN lain tak boleh reach.
❓ "Which technology separates voice and data traffic on the same switch port?" — Jawapan?
→ Voice VLAN (802.1Q)
Satu port boleh handle dua VLAN — data VLAN untuk PC dan voice VLAN untuk phone, guna 802.1Q tagging.
❓ "Desktops cannot see traffic from other devices. Which device to use?" — Router, Switch, atau AP?
→ Switch (dengan VLAN)
Switch dengan VLAN isolate traffic antara device. Router untuk routing antara network. Soalan tu dalam context network design untuk pejabat baru.
