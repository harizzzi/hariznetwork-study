FHRP — First Hop Redundancy Protocol

HSRP · VRRP · GLBP · Gateway redundancy

FHRP
First Hop Redundancy Protocol
🎯 SELALU KELUAR
🏪 Analogi — Kedai Dengan 2 Cashier
Bayangkan kau masuk kedai, ada 2 cashier — cashier A dan cashier B. Biasanya kau pergi cashier A. Tapi kalau cashier A pengsan, cashier B ambil alih automatik tanpa kau perlu buat apa-apa.

Dalam network: PC kau ada satu default gateway (contoh 192.168.1.1). Kalau router tu rosak, traffic kau stuck. FHRP selesaikan masalah ni dengan bagi dua router share satu Virtual IP — bila primary router fail, secondary ambil alih dalam masa beberapa saat, tanpa PC kau perasan.
✅
Bagi PC satu gateway address sahaja, tapi ada 2 router backup sebenarnya
✅
Guna Virtual IP (VIP) — IP yang dikongsi antara router-router dalam group
✅
Primary router fail → secondary ambil alih VIP automatik, tanpa tukar config PC

HSRP
Hot Standby Router Protocol. Cisco proprietary. Paling lama ada.
VRRP

🎯 EXAM TIP
"Which technology uses a VIP to provide gateway redundancy?" → FHRP. "Which protocol allows automatic failover for active-passive firewall pair?" → VRRP. "Open standard FHRP" → VRRP (bukan HSRP — HSRP adalah Cisco je).

LLDP & CDP — Discovery Protocols

Link Layer Discovery Protocol · Cisco Discovery Protocol

LLDP
Link Layer Discovery Protocol
🎯 KADANG KELUAR
📡 Analogi — Jiran Kenalkan Diri
Bayangkan kau pindah ke kawasan baru. Jiran sebelah ketuk pintu dan cakap: "Hai, saya Pak Ali, saya tinggal di No. 5, nombor telefon saya 012-xxx". Kau terus tahu siapa jiran kau tanpa perlu keluar jalan-jalan.

LLDP buat benda sama dalam network — setiap device announce dirinya kepada device yang connected terus: "Aku switch X, port Gi0/1, VLAN 10, model ABC". Network admin boleh tahu topologi network tanpa pergi check satu-satu secara fizikal.
✅
Discover device yang connected terus (directly connected) sahaja — bukan seluruh network
✅
Info yang dikongsi: hostname, port, VLAN, IP address, model device
✅
Open standard (IEEE 802.1AB) — semua vendor support, tak kira Cisco, HP, Juniper
LLDP
Open standard — semua vendor. Guna bila multivendor environment. IEEE 802.1AB.
CDP
Cisco Proprietary — Cisco device je. Bagi info lebih detail tapi Cisco-only.
🎯 EXAM TIP
"Technician nak tahu switch dan port mana VoIP phone connected" → LLDP. "Multivendor environment, nak document network" → LLDP (bukan CDP). LLDP hanya nampak directly connected — nak nampak semua, kena log in satu-satu dan run LLDP dari tiap device.

Routing Protocols — BGP, OSPF, EIGRP, RIP

Router guna protokol ni untuk cari jalan terbaik hantar packet

Routing Protocol
Apa fungsi dia sebenarnya?
🗺 Analogi — Waze Untuk Router
Router adalah macam driver yang nak hantar barang dari KL ke Penang. Dia ada banyak jalan pilihan. Routing protocol adalah macam Waze — dia kumpul info tentang semua jalan (router lain), kira jalan mana paling cepat/pendek, dan update secara automatik bila ada jalan tutup (router fail).

Tanpa routing protocol, admin kena manually bagitau router "jalan ke Penang ikut sini" — bayangkan buat tu untuk ribuan destination. Dengan routing protocol, router belajar sendiri jalan mana nak ambil.
BGP
Border Gateway Protocol
🎯 SELALU KELUAR
🌏 Analogi — Diplomatic Relations Antara Negara
BGP adalah routing protocol untuk internet. Bayangkan setiap ISP (Maxis, TIME, Celcom, Google, Amazon) adalah sebuah negara. BGP adalah perjanjian diplomatik antara negara-negara tu — mereka bersetuju untuk saling kongsi maklumat route.

Setiap ISP ada Autonomous System (AS) number sendiri — macam nombor negara. BGP hantar info: "Untuk reach Google (AS 15169), lalui Maxis (AS 9534) dulu".
🌐
Internet routing — guna antara ISP dan autonomous systems. Satu-satunya EGP.
📋
Pilih path guna attribute list (AS path, MED, local preference) — bukan jarak/speed
🔢
Setiap network ada Autonomous System (AS) number — macam ID organisasi
🎯 EXAM TIP
"Connect different ISP" / "internet routing" / "exchanging routes with different ISP" → BGP. "Attribute list for path selection" → BGP. BGP = satu-satunya EGP (External Gateway Protocol).
OSPF
Open Shortest Path First
🎯 SELALU KELUAR
🗺 Analogi — GPS Yang Kira Jarak Sebenar
OSPF adalah macam GPS yang kira jarak sebenar ke destination. Dia tengok semua jalan yang ada, kira mana yang paling pendek (shortest path) guna Dijkstra's algorithm, dan pilih tu.

Kalau ada jalan yang tutup (router fail), OSPF update peta dengan cepat (fast convergence) dan cari jalan lain. Semua router dalam network share peta yang sama — link-state map.
⚡
Fast convergence — update route dengan cepat bila ada perubahan
📐
Guna Dijkstra's algorithm — kira shortest path berdasarkan bandwidth/cost
📡
Hantar update guna multicast (224.0.0.5/6) — bukan broadcast, lebih efficient
🎯 EXAM TIP
"Dijkstra's algorithm" → OSPF. "Shortest path / fast convergence" → OSPF. "Interior gateway, open standard" → OSPF. OSPF guna multicast, bukan broadcast.
EIGRP
Enhanced Interior Gateway Routing Protocol
🎯 SELALU KELUAR
🏎 Analogi — GPS Cisco Dengan Backup Route Ready
EIGRP macam GPS yang bukan je tahu jalan utama (Successor), tapi dah siap sedia jalan backup (Feasible Successor) kalau jalan utama jam atau tutup. Bila jalan utama fail, dia terus switch ke backup tanpa kira semula.

Ini buat EIGRP lebih laju dari OSPF dalam situasi failover tertentu. Tapi kelemahannya — ini produk Cisco sahaja, tak boleh guna dengan router lain vendor.
🎯
Successor = jalan utama terbaik. Feasible Successor = backup route yang dah pre-computed
⚖️
Unequal cost load balancing — boleh guna beberapa jalan dengan kos berbeza serentak
🏢
Cisco proprietary. Guna port 88. Algorithm: DUAL (Diffusing Update Algorithm)
🎯 EXAM TIP
"Successor + Feasible Successor" → EIGRP. "Unequal cost load balancing" → EIGRP. "DUAL algorithm" → EIGRP. Port 88 → EIGRP.
RIP
Routing Information Protocol
📌 TAHU SAHAJA
🚶 Analogi — Kira Bilangan Traffic Light, Bukan Jarak
RIP adalah routing protocol paling simple dan lama. Dia pilih jalan berdasarkan bilangan hop (router) yang perlu dilalui — macam kira berapa traffic light nak lalu, bukan berapa jauh jarak sebenar.

Masalahnya: jalan yang ada 3 hop tapi laju 10Gbps dianggap sama dengan jalan 3 hop yang perlahan 1Mbps. RIP tak nampak speed — dia nampak hop count je. Max 15 hop.
🔢
Metric = hop count sahaja. Maximum 15 hops — lebih dari 15 = unreachable
🐢
Slowest convergence — lambat update bila ada perubahan. Legacy protocol.
🎯 EXAM TIP
"Hop count metric" → RIP. "Maximum 15 hops" → RIP. "Slowest convergence" → RIP. Soalan exam jarang tanya pasal RIP secara mendalam — biasanya compare je.

STP — Spanning Tree Protocol

Prevent loop dalam network switch

STP
Spanning Tree Protocol
🎯 SELALU KELUAR
🔄 Analogi — Dua Jalan Pergi Sekolah, Tapi Tutup Satu
Ada dua jalan dari rumah ke sekolah. Kalau kau guna kedua-dua jalan serentak, kau akan pusing-pusing tak habis — pergi jalan A, balik jalan B, pergi jalan A lagi... loop tak terhenti.

STP selesai masalah ni — dia block satu jalan (port) untuk elak loop. Kalau jalan utama rosak, STP buka semula jalan yang tadi diblock sebagai backup.

Dalam switch network: bila ada redundant links (dua kabel antara dua switch), STP block satu untuk elak broadcast storm / infinite loop.
🛡
Prevent Layer 2 loop — broadcast storm yang boleh crash network
👑
Pilih Root Bridge — switch yang jadi pusat topologi. Switch dengan Bridge ID terendah menang.
🔢
Default STP priority: 32768. Turunkan priority untuk buat switch jadi root bridge.
STP (802.1D)
Original. Slow convergence — 30-50 saat untuk converge bila ada perubahan.
RSTP (802.1W)
Rapid STP. Fast convergence — dalam beberapa saat. Guna dalam network moden.
🎯 EXAM TIP
"Network ada potential loop scenarios" → enable STP. "Default STP priority" → 32768. "Root bridge determination" → switch dengan Bridge ID terendah (priority + MAC). "Slow convergence → configure RSTP".

LACP & LAG — Link Aggregation

Gabung beberapa kabel jadi satu logical link

LACP / LAG
Link Aggregation Control Protocol / Link Aggregation Group
🎯 KADANG KELUAR
🛣 Analogi — Jalan Biasa vs Lebuh Raya 4 Lorong
Kau ada satu jalan 1 lorong antara dua bandar — maximum 100 kereta/jam. Nak tambah capacity, kau bina 3 lorong lagi di sebelah, jadi 4 lorong serentak — sekarang boleh 400 kereta/jam.

LACP buat benda sama dengan kabel network — gabung 4 kabel 1Gbps jadi satu logical link 4Gbps. Switch nampak macam satu connection je tapi sebenarnya ada 4 kabel running serentak. Kalau satu kabel putus, 3 lagi masih jalan.
⚡
Tingkat bandwidth — 4 × 1Gbps = 4Gbps logical link
🛡
Redundancy — satu kabel putus, link masih up dengan bandwidth berkurang
🔄
Prevent STP dari block redundant links — LAG nampak macam satu link, STP tak block
LAG
Link Aggregation Group — nama untuk group kabel yang digabung. Port-channel, EtherChannel.
LACP (802.3ad)
Protokol yang negotiate dan manage LAG secara automatik. Open standard.
🎯 EXAM TIP
"Connect dua switch dengan redundant links untuk tingkat bandwidth" → Link Aggregation. "40Gbps switch tapi ISP bagi 40Gbps, switch port only 10G" → guna Link Aggregation (4 × 10G = 40G). "Redundant connection between switches without STP blocking" → LACP/LAG.

QoS & CoS — Traffic Priority

Bagi priority kepada traffic yang penting

QoS
Quality of Service
🎯 SELALU KELUAR
🏥 Analogi — Kaunter Kecemasan Hospital
Bayangkan hospital dengan kaunter biasa dan kaunter kecemasan (emergency). Pesakit kritikal tak perlu beratur — mereka dapat layanan terus. Orang demam biasa perlu tunggu.

QoS buat benda sama dalam network — VoIP calls dan video conference adalah "pesakit kecemasan" yang dapat bandwidth dan priority lebih. File download yang tak urgent tunggu belakang. Ini elakkan suara VoIP jadi putus-putus (jitter) walaupun network congested.
📞
VoIP dan video dapat priority tinggi — sensitive to delay dan jitter
⬇️
File download, backup dapat priority rendah — boleh tunggu
🔧
Configure pada router atau switch — mark traffic dengan DSCP atau CoS value
QoS
Quality of Service — broad term untuk semua mechanism yang manage traffic quality. Layer 3 (IP header).
CoS
Class of Service — marking traffic priority kat Layer 2 (Ethernet frame). Guna dalam VLAN/trunk.
🎯 EXAM TIP
"VoIP voice quality poor when users upload large files" → configure QoS on router. "Video conference buffering at satellite office" → first check QoS. "Prioritize network traffic based on type" → QoS.

Singkatan Lain Yang Selalu Confuse

SDN · NFV · VXLAN · SAN · CDN · IPAM

SingkatanMaksudAnalogi MudahExam Clue
SDN Software Defined Networking Otak (control plane) dipisah dari otot (data plane) — network dimanage via software dari satu tempat "Centralized network management via software"
SD-WAN Software Defined WAN Manage semua WAN connections (MPLS, broadband, 4G) dari satu dashboard. Zero-touch provisioning = auto setup. "Zero-touch provisioning", "multiple WAN links"
NFV Network Functions Virtualization Jalankan router, firewall, load balancer sebagai software (VM) bukan hardware dedicated "Implement routing in hypervisor" → NFV
VXLAN Virtual Extensible LAN VLAN tapi boleh stretch merentasi internet/L3 network — untuk data center dan cloud "Extend VLAN across L3", "data center interconnect"
CDN Content Delivery Network Copy website/video ke server dekat dengan user — macam kedai 7-Eleven, ada kat mana-mana "Reduce latency for geographically dispersed users"
SAN Storage Area Network Network khas untuk storage — macam harddisk besar yang dikongsi dalam network laju "High speed shared storage", "FC connection"
IPAM IP Address Management Sistem untuk track dan manage semua IP dalam network — supaya tak ada duplicate atau IP habis "Track and manage IP addresses"
MIB Management Information Base Kamus untuk SNMP — file yang define apa yang boleh dimonitor dari sesuatu device "Import MIB" → SNMP cannot understand device status
AAA Authentication, Authorization, Accounting WHO are you (auth) → WHAT can you do (authz) → WHAT did you do (accounting) RADIUS dan TACACS+ implement AAA framework
SCADA Supervisory Control and Data Acquisition System monitor dan control kilang, power plant, water treatment — OT (Operational Technology) "Industrial equipment", "OT environment"
